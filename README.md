# proyecto-innovatech
# El presente documento muestra el diseño, configuración e implementación de una arquitectura cloud en Amazon Web Services (AWS) para la empresa Innovatech Chile, en el marco de la Evaluación Parcial N°1 de la asignatura Introducción a Herramientas DevOps.
El contexto del caso plantea que Innovatech Chile necesita migrar su infraestructura on-premise a AWS usando una estrategia Lift & Shift. Esta estrategia, también conocida como Rehost, consiste básicamente en tomar las aplicaciones y sistemas tal como están y moverlos a la nube sin hacer grandes cambios en el código ni en la arquitectura. Es la forma más directa de migrar, y para una empresa que está dando sus primeros pasos en cloud, tiene bastante sentido.
El objetivo concreto es levantar una prueba de concepto (POC) con una arquitectura de tres capas: Frontend, Backend y Data. El Frontend es la única capa expuesta a internet, mientras que el Backend y la capa de datos operan en subredes privadas, siguiendo el principio de mínimo privilegio. Además, se considera que en una segunda fase el sistema deberá migrar completamente a contenedores Docker, por lo que las instancias ya vienen preparadas con las herramientas necesarias.
